I. Endereços ipv6 são compostos de 8 blocos de
16 bits. Cada bloco denomina-se duo-octeto

  0000:0000:0000:0000:0000:0000:0000:0000
  ffff:ffff:ffff:ffff:ffff:ffff:ffff:ffff

II. Cada bloco é representado por 4 dígitos
hexadecimais

  0123:4567:89ab:cdef:0000:0000:0000:0000

III. Cada dígito hexadecimal representa quatro bits

  nibble    f     4 bits  1111
  byte      ff    8 bits  (1111)•2
  word      ffff  16bits  (1111)•4

  00 = 0
  ff = 255

  Um endereço ipv6 possui:
  32 caracteres hexadecimais (nibbles)
  16 bytes (octetos)
  8 words

IV. Zeros à esquera são omitíveis

  2001:0DB8:00AD:000F:0000:0000:0000:0001
  2001:DB8:AD:F:0:0:0:1

V. O ipv6 é humilde, possui apenas (undecilhões) 340.282.366
.920.938.463.463.374.607.431.768.211.456 endereços.

  Macro alocação realizada em partes pela
  IANA através dos três primeiros bits do ipv6.
      ↓↓↓
BIN-> 0010 XXXX XXXX XXXX HEX-> 2XXX
BIN-> 0011 XXXX XXXX XXXX HEX-> 3XXX

  NOTA: Considere o bit menos significativo,
  identificado pelo X maiusculo,
  como sendo zero ou um na tabela abaixo.

  HEX BIN  DESCRIPTION
  0-1 000X unspecified, loopback, embedded IPV4 addresses
  2-3 001X global unicast addresses (ALOCAÇÃO ATUAL PELA IANA)
  4-5 010X FUTURE
  6-7 011X FUTURE
  8-9 100X FUTURE
  A-B 101X FUTURE
  C-D 110X FUTURE
  E-F 111X link-local, Unique-local, Multicast addresses

Correntemente todo endereço ipv6 inicia com 2 ou 3

VI. Há três tipos principais de endereços

  Unicast, Multicast e Anycast e subdivisões.

  (UNICAST) - identifica uma interface de rede ipv6
  habilitada.
    (GLOBAL UNICAST) 2000::/3
      Endereço acessível a nível global e distribuido para
      todas instituições resposáveis pelo planeta.
    (LINK-LOCAL) fe80::/10
      Identificador autoconfigurado da interface de rede
      criado a partir do endereço MAC.
    (LOOPBACK) ::1/128
      Interface de rede lógica da própria máquina.
      Famosa 127.0.0.1 no ipv4
    (UNSPECIFIED) ::/128
      Usado por sistemas operacionais quando não há qualquer
      endereço válido e processos como o DHCP. Roteadores
      não roteiam isto.
      0.0.0.0/32 no ipv4
    (UNIQUE LOCAL) fc00::/7
      Tipo especial de endereço global único ipv6 com certas
      caracteristicas. Vide capítulo VIII
    (EMBEDDED IPV4) ::/80
      Simplesmente uma emulação do ipv4 no ipv6
      128-80 = 48bits

  (MULTICAST) - identificador de camada de rede para conjunto
  de interfaces pertecentes a diferentes nós ipv6 habilitados.
    (WELL-KNOWN) ff00::/12
      Multicasts do legados do ipv4
    (TRANSIENT) ff10::/12
    (SOLICITED-NODE) ff02:0:0:0:0:1:ff00/104
      Multicasts e pseudo-broadcast direcionado

  (ANYCAST) - identificador de camada de rede para um conjunto
  de interfaces pertecentes a diferentes nós ipv6 habilitados.

  broadcast - d e s c a r t a d o
    Funcionalidade broadcast é implementada usando
    endereços MULTICAST.

VII. Como seria ipv4 representado por hexadecimais no estilo ipv6?

  ipv4              | ipv4 hexadecimal  | estilo duo octeto ipv6
  192.168.0.0/24    | C0.A8.00.0/24     | C0A8:0000/24
  192.168.0.1/24    | C0.A8.00.1/24     | C0A8:0001/24
  192.168.0.254/24  | C0.A8.00.FE/24    | C0A8:00FE/24

VIII. Como funciona os prefixos ?

  O IPV6 mantêm estrutura REDE,HOST/XXX
  Com algumas difereças que serão sucintamente
  apontadas abaixo.

  1.UNICAST
  Identificação de uma interface.
  Comunicação de um para um.

  1.1 GLOBAL UNICAST 2000::/3 ou 3000::/3------------------

  Este endereço existe em nível PLANETÁRIO !

  a. prefixo de roteamento global       "provider"
  b. identificador de subrede           "site"
  c. identificador de interface de rede "host"

  MSB                R H                LSB
     a------------b---| c------------------
  ↓↓↓                 |
  0020:0000:0000: 0000|:0000:0000:0000:0000 /3
  0030:0000:0000: 0000|:0000:0000:0000:0000 /3

  Atualmente a IANA aloca endereços do prefixo 2000::/3
  para os provedores regionais. Exemplo: parte deste espaço
  é alocado para American Registry for Internet Numbers que,
  por sua vez aloca sub-partes deste espaço de endereços
  2001:18::/23 para Internet Serviçe Providers e grandes
  clientes como uma Microsoft® da vida.

  IANA 2000::/3
    ARIN 2001:18::/23
      ISP 2001:18B1::/32
        CLIENTE 2001:18B1:1::/48
      MICROSOFT 2001:18B1:2::/48

  Perceba a capacidade de agregação que o ipv6 possui.
  Perceba também que a capacidade de possuir um endereço
  ipv6 depende de como os orgãos (Nic.br no brasil)
  e provedoras de internet atuem ativamente.

  1.1-------------------------------------------------------

  1.2 LINK-LOCAL FE80::/10 ---------------------------------

  Tipo especial de unicast que é AUTOCONFIGURADO em qualquer
  interface de rede usando uma combinação do prefixo link-
  local (primeiros 10 bits = 1111 1110 10XX) e o endereço
  MAC da interface de rede.

  ↓↓↓
  FE8x:
  FE9x:
  FEAx:
  FEBx:

  1111 1110 10|xx 

  Onde dois bits menos significativos terceiro nibble,
  faz parte de um espaço de 54 bits.

  PREFIXO REDE IDENTIFICADOR_INTERFACE
  10 bits + 54 bits + 64 bits
  64 bits + 64 bits

  Este endereço permite que nós acoplados a um link em
  comum possam comunicar-se entre si.
  Torna DHCP ou intervenção manual opcional em rede local
  (exceto nos casos em que a placa de rede pirata que
  compraste na internet possui, por azar, um MAC indêntico
  ao da tua placa).

  Casos de uso:
    Comunicar ponta a ponta com outra máquina na rede local.
    Jogar Counter-Strike 1.6 localmente com amigos.
    Testar tua API REST rodando no PC atráves do teu
    celular.

  Exemplo de endereço MAC sem significado (camada 2):
  FF:FF:FF:FF:FF:FF

  1.2-------------------------------------------------------

  1.3 LOOPBACK ::1/128--------------------------------------
    Identifica uma interface lógica sem representante físico
    que está sempre UP e rodando. Comumente é utilizado por
    desenvolvedores para testar APIs simples e por usuários
    comuns para acessar serviços rodando em suas próprias
    máquinas.
  1.3-------------------------------------------------------

  1.4 UNIQUE LOCAL fc00::/7---------------------------------
    É globalmente único, porém tem especificidades:

    -Possui prefixo global único similar a endereço global
    unicast. Se acidentalmente vazado para fora de uma
    instituição não conflita com outros endereços ipv6
    -Sua estrutura é bem conhecida, permitindo filtragem
    fácil.
    -Permite interconexão de sites sem criação de nenhum
    conflito.
    -Depende de espaço de endereço de provedor de internet
    independente. Caso contrário conflitos podem ocorrer por
    uso indevido de outros espaços de endereços alocados.
    -Aplicações tratam tais endereços como ipv6 global
    regular (indistigue).

    Os roteadores da Internet filtram quaisquer rotas de
    entrada ou saída do IPv6 local. A estrutura de um
    endereço local exclusivo é mostrada abaixo:
    
    PREFIXO GLOBAL_ID SUBNET_ID INTERFACE_ID
    FC00:   42bits    16bits    64bits

  1.4-------------------------------------------------------

  1.5 EMBEDDED IPV4-IN-IPV6 ::FF:FF:FF:FF-------------------
    Endereço unicast com zeros nos primeiros 96 bits
    (128-96 = 32)
    Trata-se de uma sub-camada de retrocompatibilidade com o
    largamente utilizado ipv4.

  1.5-------------------------------------------------------

  2. MULTICAST

  ff00::/8

  Um nó manda pacotes para multiplos destinos
  simultâneamente (De um para Muitos).

  Os destinos são um conjunto de intefaces de rede
  conhecidos como Grupo Multicast.

  Os primeiros 8 bits são 1 (dois F's).
  Equivale a 224.0.0.0/4 multicast do ipv4.
  
  Todo pacote enviado a um multicast possui um endereço
  unicast de origem. Endereço multicast não pode ser endereço
  origem de um pacote.

  Broadcast não existe no ipv6, multicast grupos multicast
  especiais cumprem este papel.

  2.1 WELL-KNOWN MULTICAST ADDRESSES ff00::/12--------------

    Herdado do ipv4 e ainda importante para uso especial

    FF02::1     Todos endereços nós
    FF02::2     Todos endereços roteadores
    FF02::4     Roteadores DVMRP
    FF02::5     Todos roteadores OSPFv3
    FF02::6     Roteadores OSPFv3 desginados
    FF02::A     Todos roteadores EIGRP (IPV6)
    FF02::D     Todos roteadores PIM
    FF02::12    VRRP
    FF02::16    Todos roteadores MLDv2-capable
  2.1-------------------------------------------------------
    
  2.2 SOLICITED-NODE MULTICAST ADDRESS FF02::1:FF00:0/104---
    Um endereço multicast de nó solicitado é um tipo
    especial de multicast IPv6. É usado como uma abordagem
    mais eficiente para a entrega de transmissão do IPv4. Um
    endereço multicast de nó solicitado é gerado
    automaticamente usando um unicast IPv6 de uma interface.

    EXEMPLO:
    Unicast address
    2001:0000:0000:0000:0000:0000:12|34:ABCD
    Multicast group
    FF02:0000:0000:0000:0000:0000:FF|34:ABCD

    Diferente do broadcast ipv4, onde todos nós respondiam,
    apenas o nó alvo responde processando a requisição.

  2.2-------------------------------------------------------

  3. ANYCAST

  Um endereço anycast é um identificador de camada de rede
  normalmente atribuído a mais de uma interface (um conjunto
  de interfaces), pertencente a diferentes nós habilitados
  para IPv6.  Pacotes enviados para um endereço anycast são
  entregues na interface "mais próxima" identificada por
  esse endereço. "Mais próximo" normalmente significa aquele
  com a melhor métrica de roteamento de acordo com o
  protocolo de roteamento IPv6. 

  Os endereços Anycast são alocados a partir do espaço de
  endereços unicast, portanto, eles são indistinguíveis de
  endereços unicast globais. Configurar o mesmo endereço
  unicast para mais de uma interface faz com que seja um
  endereço anycast. Os dispositivos que possuem um endereço
  anycast atribuído devem ser explicitamente configurados
  para reconhecer que o endereço é usado para comunicação
  anycast.
